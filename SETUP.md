# Local Development Setup

Instructions for getting this project running on a new Mac from scratch.

---

## Prerequisites

### Node.js

This project requires Node.js 20+. Install via [nvm](https://github.com/nvm-sh/nvm) (recommended):

```bash
nvm install 20
nvm use 20
```

Or download directly from [nodejs.org](https://nodejs.org).

### Turso CLI

Used to manage the Turso (libSQL) database.

```bash
brew install tursodatabase/tap/turso
turso auth login
```

### Stripe CLI

Used to forward webhooks to your local server during development.

```bash
brew install stripe/stripe-cli/stripe
stripe login
```

---

## Clone and Install

```bash
git clone https://github.com/bkasanoff/kasanoff-alive.git
cd kasanoff-alive
npm install
```

## Environment Variables

Create a `.env.local` file in the project root with the following variables. **Never commit this file.**

```
ANTHROPIC_API_KEY=
STRIPE_SECRET_KEY=
STRIPE_WEBHOOK_SECRET=
TURSO_DATABASE_URL=
TURSO_AUTH_TOKEN=
ADMIN_PASSWORD=
SESSION_SECRET=
NEXT_PUBLIC_BASE_URL=
SENDGRID_API_KEY=
```

### Variable descriptions

| Variable | Description |
|---|---|
| `ANTHROPIC_API_KEY` | Anthropic API key for Claude. Get from console.anthropic.com. |
| `STRIPE_SECRET_KEY` | Stripe secret key. Use the **test mode** key (`sk_test_...`) for local dev. |
| `STRIPE_WEBHOOK_SECRET` | Webhook signing secret. Generated by the Stripe CLI when you run `stripe listen` (see below). |
| `TURSO_DATABASE_URL` | libSQL connection URL for the **dev** Turso database (see warning below). |
| `TURSO_AUTH_TOKEN` | Auth token for the **dev** Turso database (see warning below). |
| `ADMIN_PASSWORD` | Password for any admin-only routes. Choose anything locally. |
| `SESSION_SECRET` | Secret used to sign iron-session cookies. Use a long random string (40+ chars). |
| `NEXT_PUBLIC_BASE_URL` | Set to `http://localhost:3000` for local development. |
| `SENDGRID_API_KEY` | SendGrid API key for sending magic link emails (can use a placeholder locally if not testing email). |

### ⚠️ TURSO_DATABASE_URL and TURSO_AUTH_TOKEN must always point to the dev database

Never use the production Turso database URL or token in `.env.local`. The dev database is:

```
TURSO_DATABASE_URL=libsql://explorations-dev-bkasanoff.aws-us-east-1.turso.io
```

Get the corresponding `TURSO_AUTH_TOKEN` from Bruce or from `turso db tokens create explorations-dev`. The production database (`explorations-bkasanoff`) is only used by Vercel in the deployed environment and should never be pointed at locally.

## Database Setup

Run the setup script to create all tables in the dev database:

```bash
npm run db:setup
```

This creates: `customers`, `purchases`, `sessions`, `messages`, and `magic_links`.

---

## Running Locally

### Start the dev server

```bash
npm run dev
```

App runs at [http://localhost:3000](http://localhost:3000).

### Forward Stripe webhooks

In a separate terminal, run:

```bash
stripe listen --forward-to localhost:3000/api/stripe/webhook
```

Copy the webhook signing secret printed by this command (`whsec_...`) and set it as `STRIPE_WEBHOOK_SECRET` in `.env.local`. Restart the dev server after updating the file.

---

## Verifying the Setup

1. Visit [http://localhost:3000](http://localhost:3000) — the homepage should load.
2. Click into any exploration — you should be prompted to purchase or claim access.
3. Use a Stripe test card (`4242 4242 4242 4242`) to complete a test purchase.
4. The Stripe CLI terminal should show an incoming `checkout.session.completed` event.
5. After purchase, you should receive a magic link email and be able to start the exploration.
